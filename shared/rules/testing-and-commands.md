# Testing and Commands

Design the tests or validation steps that should be run. Do not run shell commands, scripts, test suites, or validation scripts unless the user's current preferences permit command execution or the user explicitly grants permission for this task.

Use this together with stack-specific testing guidance for framework-specific testing patterns and tool choices.

When command execution is restricted, describe the exact commands or validation steps the user may allow, but keep the command examples in stack-specific skill files.
