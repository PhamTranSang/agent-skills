# Example: Ambiguous Intake

## User input

> Improve the approval flow experience.

## Expected behavior

- Ask up to 3 focused questions first
- Do not invent the scope
- State assumptions explicitly if the user wants a draft before answering

## Good clarification questions

- Which user role owns the approval action?
- What exact step is confusing today?
- What is the success criterion for phase 2?

## If the user wants a draft anyway

- Problem summary
- Assumptions
- Candidate scope
- Risks
- Open questions
- Handoff notes for `engineering-mentor`
