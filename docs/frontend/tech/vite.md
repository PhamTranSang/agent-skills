# Vite

Use this file when the task is about Vite configuration, aliases, plugins, dev/build behavior, or app-versus-library mode.

## Tooling

- Follow existing Vite app or library mode.
- Keep aliases aligned with TypeScript config.
- Avoid adding plugins unless the framework or feature requires them.
- Inspect env handling, test config, and plugin setup before changing the build.

## Practical Fit

- Prefer Vite for fast SPA workflows, small-to-medium product apps, design system sandboxes, or component libraries.
- Keep Vite focused on build/runtime ergonomics rather than application architecture decisions.
