# Tailwind CSS

Use this file when the task is about Tailwind setup, class strategy, theme tokens, or utility-first UI implementation.

## Usage

- Use existing Tailwind config, theme tokens, plugins, and class ordering conventions.
- Prefer design-system components or composed utility patterns when they already exist.
- Avoid arbitrary values unless they reflect a one-off design constraint that cannot be represented by tokens.
- Configure the correct content paths, PostCSS integration, framework adapter, and base stylesheet imports when setting up Tailwind.

## Fit

- Tailwind is a strong default for fast polished UI when the user wants speed and has not chosen MUI or Ant Design.
