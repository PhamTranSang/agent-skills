# Coordination With Engineering Mentor

Use `engineering-mentor` when the plan needs technical validation, implementation sequencing, trade-off analysis, or a sanity check on feasibility.

Ask `engineering-mentor` to:

- validate whether the scope is technically feasible as proposed
- identify hidden dependencies, edge cases, and risky shortcuts
- suggest the smallest coherent implementation slices
- flag tickets that should be split, renamed, or moved to a spike

Carry the mentor feedback into the final plan as:

- revised scope or sequencing
- changed priority or estimates
- new technical subtasks or spikes
- updated risks and assumptions
